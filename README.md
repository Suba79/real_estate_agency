# Сайт риэлторского агентства

Учебный Django-проект для агентства недвижимости.

На сайте доступна главная страница со списком квартир, фильтрами и административная панель для работы с объявлениями, собственниками, жалобами и лайками.

Проект сделан в рамках урока Devman.

## Возможности

- просмотр списка квартир;
- фильтр по городу, цене и новостройкам;
- админка для управления квартирами;
- поиск и фильтры в админке;
- хранение собственников квартир в отдельной модели;
- связь квартир с собственниками;
- хранение жалоб пользователей;
- хранение лайков квартир;
- нормализация телефонных номеров владельцев.

## Как запустить проект

### 1. Скачайте код

```bash
git clone https://github.com/Suba79/real_estate_agency.git
cd real_estate_agency
```

### 2. Создайте виртуальное окружение

Для Windows PowerShell:

```powershell
py -m venv .venv
.\.venv\Scripts\activate
```

Для Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Создайте файл `.env`

Создайте файл `.env` рядом с `manage.py`:

```env
DEBUG=True
SECRET_KEY=dev-secret-key
ALLOWED_HOSTS=127.0.0.1,localhost
DATABASE=sqlite:///db.sqlite3
```

### 5. Подготовьте базу данных

Если у вас есть готовая тестовая база `db.sqlite3`, положите её рядом с `manage.py`.

Затем примените миграции:

```bash
python manage.py migrate
```

### 6. Создайте администратора

```bash
python manage.py createsuperuser
```

### 7. Запустите сайт

```bash
python manage.py runserver
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

Админка доступна по адресу:

```text
http://127.0.0.1:8000/admin/
```

## Переменные окружения

Проект использует переменные окружения из файла `.env`.

Доступные переменные:

- `DEBUG` — режим отладки. Для локального запуска можно указать `True`.
- `SECRET_KEY` — секретный ключ Django-проекта.
- `ALLOWED_HOSTS` — список разрешённых хостов через запятую.
- `DATABASE` — адрес базы данных. Для локального SQLite можно использовать `sqlite:///db.sqlite3`.
